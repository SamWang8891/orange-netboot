Setup Guide

1 Server Setup (one-time)

Install NFS on the host and start Docker containers:

# Install NFS server on the host
sudo ./server/setup-host.sh

# Start TFTP + Web UI containers
docker compose up -d
Docker runs TFTP (:69) and this Web UI (:8080). NFS runs directly on the host.
2 Router DHCP Configuration

Tell your router to point booting devices to this server's TFTP.

Find your router's DHCP settings (sometimes under LAN / Network / Advanced):

TFTP Server{{ server_ip }}
Enable Network Booting☑️ Checked
Next Server{{ server_ip }}
Common routers:
  • pfSense / OPNsense: Services → DHCP Server → [LAN] → TFTP / Network Booting
  • OpenWrt: Network → DHCP and DNS → TFTP Settings
  • MikroTik: IP → DHCP Server → Options (next-server, boot-file-name)
  • dnsmasq: dhcp-boot=tag:*,pxelinux.0,{{ server_ip }}
Leave all filename/boot-file fields blank. U-Boot's PXE boot will find the config automatically by MAC address.
3 Deploy a Node (run on host)

Extract rootfs + kernel from an Armbian image and set up NFS:

# Basic — deploy without MAC (use pxelinux.cfg/default)
sudo ./server/deploy-rootfs.sh Armbian.img mynode

# With MAC — creates per-board PXE config
sudo ./server/deploy-rootfs.sh Armbian.img mynode 02:81:d7:f0:44:df

# With MAC + specific board DTB
sudo ./server/deploy-rootfs.sh Armbian.img mynode 02:81:d7:f0:44:df sun8i-h3-orangepi-pc.dtb

This creates /srv/nfs/mynode/ (rootfs), data/tftp/mynode/ (kernel+DTB), and data/tftp/pxelinux.cfg/01-xx-xx-xx-xx-xx-xx (PXE boot config).

Don't know the MAC yet? Deploy without it, boot the board, find MAC in your router's DHCP leases, then set it here in the web UI.

4 Prepare SD Card

Flash a stock Armbian image to an SD card. No modifications needed!

# Use dd, balenaEtcher, or Raspberry Pi Imager
sudo dd if=Armbian_bookworm_orangepione.img of=/dev/sdX bs=4M status=progress
U-Boot's built-in PXE boot will automatically find the TFTP server via DHCP. The SD card just provides U-Boot — everything else comes from the network.
5 Boot & Verify
  1. Insert SD card into the board
  2. Connect ethernet
  3. Connect HDMI (optional — shows kernel boot messages)
  4. Power on
  5. Wait ~30 seconds for boot
  6. SSH in: ssh root@<ip-from-router-leases>

Default Armbian root password: 1234 (prompted to change on first login)

How It Works
  1. Board boots U-Boot from SD
  2. U-Boot sends DHCP request
  3. Router replies with IP + TFTP server
  4. U-Boot requests pxelinux.cfg/01-<mac> via TFTP
  5. Config specifies kernel, initrd, DTB, NFS root
  6. Kernel boots, mounts NFS rootfs
  7. Linux starts normally
Troubleshooting

Board doesn't boot at all?
Check router DHCP leases — is the board getting an IP?

No TFTP traffic?
sudo tcpdump -i any port 69 -n

Kernel loads but NFS fails?
Check NFS export: sudo exportfs -v
Check firewall: ports 111, 2049, 20048

Kernel panic "no init"?
Missing initrd. Re-run deploy-rootfs.sh.
